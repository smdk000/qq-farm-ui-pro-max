# 系统设计：优化 Admin 界面布局

## 1. 架构与组件
- **`UserInfoCard.vue`**: 修改现有的垂直流式布局。当前的卡片上方是 `bg-gradient-to-r` 显示欢迎语、状态标签、角色，右侧有退出按钮；下方 `p-6` 区域包含三行文本和两个全宽按钮。
  - **新设计**：取消下方的大面积空白区，将卡密信息、到期时间和按钮等，全部融入到一个更加紧凑的高度中，或全部放置于卡片头部。
  - **实现方案**：
    1. 改为一行的 Flex 布局架构。
    2. 左侧展示头像与账号等级（或将其保持紧连）。
    3. 中间展示卡密详情与到期时间，采用 Badge（徽章）和紧凑文本 `text-xs` 或 `text-sm`。
    4. 右侧保留“续费”、“改密”和“退出登录”按钮，改为并排的小按钮。
    5. 当有过期提示（Warning）时，将提示横幅放在头部下方，紧凑展示。

- **`Dashboard.vue`**:
  - `div.grid.grid-cols-1.gap-4.lg:grid-cols-3.sm:grid-cols-2` 内部的三个卡片：
    1. *Account & Exp*
    2. *Assets & Status*
    3. *Items (Fertilizer & Collection)*
  - **实现方案**：
    1. 减少卡片的 padding 从 `p-4` 为 `p-3` 或更小。
    2. `mb-2` 减少为 `mb-1`。
    3. 提升子文字和行的对齐度，利用 Flex 排列而不是直接换行，极大地节约高度。
    4. “今日统计”外层面板高度可伸缩且拥有 `overflow-y-auto`，给它更多空间。为了使其“显示得更好”，其内部的条目高度 (`py-2` 等) 其实保持原样或略微加大，以保证美观，因为总体框架的高度省下来了。

## 2. 交互与依赖
没有任何后端接口或数据逻辑的修改，纯前端模板及样式的重构（Tailwind CSS classes）。
原有的 `showRenewModal`, `showPasswordModal`, `handleLogout` 逻辑保持完全不变。

## 3. 风险与影响
- `Dashboard` 布局的调整可能导致在非常小宽度的移动端上出现拥挤。
  - **缓解**：通过 Tailwind 的响应式前缀（`sm:`, `md:`, `lg:`）确保在移动端自动换行，保持可用的高度流式特性；在中大屏幕上采用扁平化、行内对齐来节省空间。
