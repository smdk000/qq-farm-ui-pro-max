# 任务对齐：优化 Admin 界面布局

## 1. 需求分析
- **目标**：优化 `Dashboard ` 页面和 `UserInfoCard` 组件的高度占用，使得下方内容（尤其是“今日统计”卡片）能更好地展示。
- **具体要求**：
  1. **UserInfoCard 高度优化**：将“卡密类型”、“到期时间”、“剩余时间”以及“续费”、“改密码”按钮全部上移，合并到一排或更紧凑的布局中，大幅节约该卡片的高度。
  2. **账号状态行卡片高度优化**：Dashboard 中间一行的三个状态卡片（账号经验、金币点券资产、化肥收藏点）进行紧凑化处理，减少垂直 padding 和多余的留空。
  3. **今日统计卡片更好地显示**：随着上方组件高度压缩，下方的“今日统计”可以获得更多的展示高度，同时优化一下内部的滚动排版。
  4. **约束**：保持原有的风格一致（配色、深色模式支持、圆角、组件样式）。

## 2. 界面变更说明
- **UserInfoCard**：目前分为“头部区块（带渐变背景）”和“内容区块（白色/深色）”。计划将被分离的卡密信息和操作按钮移动至头部区块内部，或者紧凑的一行信息栏，从而去除下半部分大面积空白。
- **Dashboard 三状态卡**：减小内部 `<div class="p-4">` 至 `p-3`，调整 `margin-bottom` 或 `gap` 以缩小高度，使得字体与图标之间的间隙更加紧凑。

## 3. 验收标准
- [ ] UserInfoCard 高度大幅减小，不再有多余的下方大段空白区域。
- [ ] 卡密、到期时间、剩余时间、续费按钮、改密按钮成功整合在卡片的顶部/右侧紧凑位置。
- [ ] Dashboard 三状态卡片采用更紧凑的设计，内容依旧清晰可见。
- [ ] 今日统计卡片内容展示更加充分，滚动条体验更好。
- [ ] 无功能异常，各按钮（续费、改密、退出登录）能够正常响应。
