# 标准作业程序 (SOP): 项目结构治理与自动化部署规范

## 1. 结构管控红线 (Filesystem Constraints) 🚨
为了避免项目无序庞大化与未来远端部署时遭遇隐性的数据链路丢失，任何开发参与者均需遵循此空间隔离基准：
- **禁止无意义污染根目录**：根目录仅允许作为**配置与入口枢纽**保留。除 `package.json`, `pnpm-workspace.yaml`, `.gitignore` 及极为轻量的核心一键部署引导壳外，严禁在此乱弹 `.sh` 或产生冗余备份。
- **文档强制沉淀 (`docs/`)**：所有操作规范说明（包含历史遗留的 `ADMIN_PASSWORD_GUIDE.md`, `FARM_SH_GUIDE.md`）、架构演进与复杂分析计划报告等汇总至该目录体系，拒绝游丝蔓延。
- **全系脚本收容 (`scripts/`)**：不再乱置 `quick-start.sh` 或密码修改运维脚本。将全部归档于 `scripts/utils/`（常驻功能辅助）与 `scripts/deploy/`（构建、连接发布服务）内分而治之，并通过顶层单入口进行统筹调用。
- **构建输出与污染品归档 (`.archives/`)**：产生的源码备份、SQL dump 以及 `qq-farm-ui-***.tar` 部署制品严禁散落周边，统一纳入私密的 `.archives/`，其必须保持常驻忽略序列，切断一切远端同步误触。

## 2. ".dockerignore" / ".gitignore" 绝对路标防护法 ⚠️
**严重防灾警告**：永远禁止使用粗暴名词作泛匹配过滤！
- ❌ **极度危险写法**：直接定义 `data`，这将引发诸如内部工程 `web/src/data`（例如植物等前端配置参数 JSON 层）在构建或镜像上下文解析时惨遭抹除！
- ✅ **防御性正确写法**：必须采用**绝对起始定界符**。只有位于特向挂载点的 `/data`、`/core/data` 或 `/logs` 方可录入屏蔽白名单。新规之后务必采用终端 `git check-ignore` 进行贯穿演练。

## 3. 完全规避性的一键部署规程 (The Zero-Interference Deploy)
为了跳出本次我们与容器环境遇到的各种“暗坑”，之后的 `deploy_prod.sh` 自动生成脚本将依附下述逻辑强力阻绝问题：
1. **源端验证与前置组装**：通过指令执行 `pnpm build` 及各项验证（杜绝打包不完整）。
2. **纯净源码传输通道**：放弃容易中断的直接远程 Pull 策略。构建同步流机制依赖精准调配的 `rsync` 排除集（筛去本地 `node_modules` 与 `/logs` 噪音）保障纯粹同步。
3. **环境免疫筑基**：针对远端环境组建（如原生 Alpine 系统 TLS 超期以及 Node C++ 包下载封锁），新构 Dockerfile 强制置入网络反解预热 —— 打入**阿里 npm 加速节点**与**系统依赖镜像包**。
4. **弹性升级与历史对接**：远端驱动采用 `docker compose up -d --force-recreate`，它将自动牵引对位并无伤承载旧存在的持久化日志目录与高危账套库，重启最终甩出实时 50 行追溯尾日志以供审定体验。

## 4. GitHub 开源双子星同步逻辑 (github-sync)
- 仅提供专门定制化排除同步 Shell 向 `/github-sync` 同步干预完毕后的核心工程。
- 内库永远需要携带 `.env.example` 与安全脱敏版 `accounts.json`，在任何迁移触发前，杜绝隐私账套库以及服务器联通特权 token 进行过线外溢！

> 💡 **项目凌乱时的紧急自救**：
> 当频繁开发导致杂碎重开，不要犹豫，直接参照 SOP 第一道防线清剿。如遇不可信的结构状态点，优先排查忽略阻断名单。此文件具有核心纠偏权。
