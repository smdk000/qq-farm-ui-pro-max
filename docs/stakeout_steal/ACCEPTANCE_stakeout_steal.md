# ACCEPTANCE: 验收标准

## 验证项

1. **设置保存与回显**
   - 前端能在“偷菜设置”中看到“蹲守偷菜”的开关与延迟秒数 (0-60秒)。
   - 切换开关与调整数值后可以被成功保存。
   - 刷新页面后能正确从后端读取最新的 `stakeoutSteal` 值并回显。

2. **预测成熟分析**
   - `analyzeFriendLands` 根据作物的种植时间与生长周期准确预判将要成熟的作物，过滤掉黑名单和无需处理的地块。

3. **蹲守挂载与隔离**
   - 在巡查 (`visitFriend`) 中能够计算时间并放入 `friendScheduler` 中作为延时任务挂起。
   - 多次扫描同一位好友时，不会引发大量重复的定时器（原有相同前缀的定时器会被清除）。

4. **蹲守动作执行**
   - 当倒计时结束：
     - 若成功：进入对应农场执行 `stealHarvest`，并且日志能够输出`[蹲守] ... 偷取成功`。
     - 若失败/未成熟：会抛出重测提示或最终放弃，并正确退出农场。

## 结论
符合技术需求，验证通过。
