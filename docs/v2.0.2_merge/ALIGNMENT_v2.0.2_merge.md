# 6A 阶段: Align (对齐) - 外部 V2.0.2 优化合并分析

## 1. 分析背景
根据用户提供的外部 `/Users/smdk000/文稿/qq/qq-farm-bot-ui-main 3` 的 V2.0.2 更新日志，以及与我们当前系统 (v3.2.7) 的深度比较，我们梳理出了需要“吸纳合并”的补丁与可跳过的交集。

## 2. 功能逐一审查

### ✅ 需要被吸纳（Port-in）的优秀功能
1. **延长有机肥施肥间隔（防风控）**: 
   - 外部版本将单次施肥循环间隔从 `100ms` 加长到了 `500ms`。
   - **行动**: 将引入此设定，并与我们系统现有的 `MAX_ORGANIC_ROUNDS=500` 双重防护结合，形成完美的安全施肥控制。
2. **账号选择逻辑优化**:
   - 外部支持在 `Accounts.vue` 中点击整张卡片（或指定区域）直接将其选中为当前激活账号。
   - **行动**: 将在我们的 `Accounts.vue` 组件上添加相映的快速聚焦逻辑，减少下拉框冗余操作。
3. **好友页面交互优化**:
   - **行动**: 在 `Friends.vue` 添加好友搜索框，同时剥离 30 秒的死板自动刷新定时器（替换为操作后的局部/手动刷新），避免浪费网络请求并导致操作打断。
4. **分离昵称和备注**:
   - 外部修复了同步时将“用户备注”(name) 被生硬替换覆盖为 “网名”(nick) 的问题。
   - **行动**: 将分离 `core/src/runtime/worker-manager.js` 中关于 `name` 与 `nick` 的更新逻辑。
5. **补充：“账号运行时间”同步系统时间的修复**:
   - 外部修复了重启机器会导致已运行时间归零或错乱的计时器 Bug。
   - **行动**: 将审查心跳及 Uptime 算法，使其记录绝对时间戳。

### 🌟 我们已经实现甚至超越的功能（跳过）
1. **修复两季作物第二季被铲除**: 我们在之前的 v2.0.4 和数据库迁移阶段，使用了经过严密测试的“收获后再测”高级状态判定算法，精准拦截，此项我们更优。
2. **修复日志筛选器选项为空不刷新的问题**: 我们已借由 `@vueuse/core` 挂载至深层 `useStorage` 彻底接管缓层，没有此 Bug 且体验更为卓越。
3. **初次登录后前后端主题统一**: 我们开放了 `/api/ui-config`，真正意义上支持壁纸与样式的鉴权前公开，处于业界领先地位。
4. **修复策略选种预览样式**: 我们已经在 `v3.2.1` 版严格排版修正完毕。
5. **微调手机端及重连机制**: 在核心通讯库底层（如 `network.js`），Diff 表明部分 PR（如 Find-fengqingwu的重连）实则已经被我们在之前的核心升级中包含。
