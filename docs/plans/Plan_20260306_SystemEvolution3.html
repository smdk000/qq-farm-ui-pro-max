二、原子级阶段任务 (Atomized Execution)

Module 1 赋视: MySQL 日志翻链 API

  • 由于内存中的保留被缩减到 300，我们需要开放 GET /api/logs 以拉取全量。
  • 实现动态组装 SQL（防注）：根据 Level、时间戳范围、UID 进行多维翻查。

Module 2 搭台: Frontend 审计台 (UI)

  • 开发 SystemLogs.vue 组件挂载至后台主界面菜单。
  • 利用 Vue 编写交互友好的日期筛选器、严重级别级联选项器和分页表组件。

Module 3 削石: 瓦解巨物 friend.js

  • 该文件承载了过多的业务域。将在 core/src/services/friend/ 下创建分形。
  • 将原先杂糅的 HTTP 封装、策略调度、结果判定拆散，依靠 Façade (门面对象) 重绑，对外部入口 0 损耗变更。

三、演进收益 (Impact Matrix)

1. 开发体验 (DX): friend.js 拆分后，下一期开发人员将不必再面对极其容易合错代码产生冲突的单体结构。

2. 客户成功 (CS): 随着多账号的横向扩展，前端 300 条 LRU 缓存只需 1 分钟就能被冲刷干净（特别是有微信被禁掉时）。现在全量查询台可随时支持故障溯源，做到“死后可对证”。
