二、执行路径拆解 (Atomized Milestones)

Task 1 (P0) MySQL 原子更新与死锁防护

目标： 隔离底层读写污染并实施高阶同步闭环。

  • 在 mysql-db.js 引入 withTransaction 闭包辅助。
  • 对核心经济和数据变更实施 BEGIN -> COMMIT 防护，错误时 ROLLBACK。

Task 2 (P1) QoS 下沉：动态 TTL 权重策略

目标： 规避一刀切拦截导致的 UI 长效白屏假死。

  • 队列入栈挂载动态 ttl: 常规接口(3s)，写入接口(15s)，抢收(无限)。
  • 超时不再直接抛掷 Error，而是传递给业务层一个静默可恢复重试的特征码。

Task 3 (P1) 战略隔离：平台策略模式解耦

目标： 剔除 friend.js 里极度难看的微信端判定硬编码代码。

  • 建立 platform/ 设计模式接口。将 QQ 版独有的偷菜，微信版的休眠统合入多态类内部隔离。主干只调用 executeFriendCycle() 通用接口。

Task 4 (P2) 冷数据入库引擎：全链条落库

目标： 补足前端由于 LRU 被限制到 300 条后丧失的问题追踪能力。

  • 新增表格专门用作 Log 归档。挂载到批量插入 Batch 队列里减少 I/O 成本。

三、演进验收与风险自评

里程碑 影响控制面
Task 1 - TCC 级事务 由于抛弃 Promise Fire-And-Forget 并强拉起 Connection 对象，这期间必须严格注意回调死锁可能阻塞 Pool (连接池枯竭) 的致命伤。
Task 2 - 权重 TTL 需修改网络层，可能会波及极少数依赖瞬间响应返回错误链的老业务，必须严格自测其无缝性。
Task 3 - 平台抽象层 代码大量重构大手术（重灾区），为了避免闪退，将逐步剥离，并严格维持单元入参出参不变。
