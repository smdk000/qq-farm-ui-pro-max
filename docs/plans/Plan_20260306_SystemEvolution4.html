2. 原理与实施步骤 (Execution Steps)

Step 1: 建立熔断隔离护城河 (Circuit Breaker)

在底层实现自动保护，不完全切断业务但暂停 API 发射池（类似网络层的短路开关）。在连续 N 次触碰预设红线后锁定容器调度。

Step 2: P0 级别路由治理 (admin.js)

admin.js 的行数由于历史版本堆积越来越长，必须将完全纯计算型的工具 API 隔离到 farm-tools-routing.js 以腾出主进程内存。

Step 3: Echarts 数据聚合大屏引入

在终端展示外，让前端提供近7天偷菜金币量与工作效能折线图，直观展示系统的赚钱能力。无需外部重量级 BI 系统。

3. 影响面评估 (Impact Analysis)

🚨 警告领域：修改路由拆分时，前端请求地址将会变更 (`=> /api/tools/...`)，可能会引发暂时的 404 错误。需要确保所有涉及前端请求的文件（比如 Vue stores 或组件）一次性被安全替换。

💡 熔断风验：熔断机制的恢复条件（Half-Open）设置必须精确，否则可能导致整个框架永久停止运行无法挂机，建议增加手动强制重启的功能按钮或通过重启容器归零状态。
