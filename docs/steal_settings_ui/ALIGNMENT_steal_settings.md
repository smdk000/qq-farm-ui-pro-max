# 独立偷菜设置页 - ALIGNMENT 文档

## 1. 目标与背景
用户提出需要一个独立的左侧菜单项「偷菜设置」，在此页面集中且优化展示「好友偷菜过滤」和「作物偷菜过滤」的功能。现存的这部分设置潜藏在长列表的 Settings.vue 中，层级较深，且对于大量的作物的勾选，交互体验不足。
- **业务价值**: 降低高阶用户精细化配置的门槛，提升面板的专业感与可玩性。
- **功能期望**: 
  - 左侧边栏新增路由入口。
  - 右侧主工作区一分为二（双 Tab），提供好友过滤和作物过滤。
  - 列表交互需要可视化程度高（含图标、等级、名称）。
  - 提供顶层的批量操作（全部偷、全不偷）和搜索过滤。
  - 设置数据接管现有的配置流并入库保存。

## 2. 现状分析
当前后端已存在完整的 `account_configs` 相关设置与表：
- `account_friend_steal_filter`: 保存 `friend_id` 和 `type` (1白2黑)。
- `account_plant_filter`: 保存 `plant_id` 和 `type` (1白2黑)。
前端在 `Settings.vue` 中获取上述数据进行渲染，但只有简单的文字 Checkbox。

## 3. 技术约束与规范
- 使用现有的 `/api/settings` 和 `/api/settings/save` 进行数据读写。
- UI/UX：遵循用户分享的意图，应用 `ui-ux-pro-max` 提倡的现代化交互卡片、状态色以及网格系统。使用 Tailwind CSS，保留系统现有的响应式框架与深色模式。
- 组件级：复用现有的 `BaseButton`、`BaseInput` 以及从全局静态列表中取游戏字典（作物 `seedId` to `name` / `image`）。

## 4. 交付验收标准
- [ ] 确保新增 `/steal-settings` 路由能正常访问且左侧高亮。
- [ ] 新页面能正确列出农场所有的作物（带图标和等级），并提供美观的网格排版。
- [ ] 搜索框能够对好友与作物列表进行客户端内存级快速文本过滤。
- [ ] 分类控制能正常工作，包括“批量偷”和“批量不偷”的业务逻辑联动。
- [ ] 修改参数后点击保存，能够将数据持久化并落库，不影响现存数据。
- [ ] （可选）隐藏 `Settings.vue` 里的原生复选框列表，转移指引到新页面。
