## 📖 内容编写准则

### 文章结构标准

每篇帮助文章应包含以下部分：

```markdown
# 文章标题

## 简介
- 简短说明文章主题
- 说明适用场景
- 预计阅读时间

## 核心概念
- 解释关键术语
- 说明基本原理
- 提供背景知识

## 操作步骤
1. 第一步...
2. 第二步...
3. 第三步...

## 配置示例
```yaml
# 配置代码示例
key: value
```

## 注意事项
- ⚠️ 需要特别注意的地方
- ❌ 常见错误
- ✅ 最佳实践

## 相关问题
- 链接到相关文章
- 常见问题解答

## 小贴士
- 💡 实用技巧
- 🎯 优化建议
```

### 写作风格指南

#### 语气风格

**推荐**：
- ✅ 友好亲切
- ✅ 专业但不生硬
- ✅ 鼓励性语言
- ✅ 使用"你"而非"用户"

**避免**：
- ❌ 命令式语气
- ❌ 过于技术化
- ❌ 消极语言
- ❌ 模糊表述

---

#### 格式规范

**标题层级**：
```
# 一级标题（文章标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（细节说明）
```

**列表使用**：
- 有序列表：步骤、顺序
- 无序列表：要点、特性

**强调方式**：
- **粗体**：重要概念、关键操作
- *斜体*：术语、引用
- `代码块`：代码、配置

---

#### 图示规范

**截图要求**：
- 分辨率：1920x1080 或更高
- 格式：PNG（无损）
- 标注：使用红色框或箭头
- 说明：截图下方添加说明文字

**图标使用**：
- 使用 Carbon Icons
- 尺寸统一（text-xl）
- 颜色符合主题
- 语义准确

```

docs/
├── HELP_CENTER_IMPLEMENTATION.md # 实施文档
├── HELP_CENTER_MAINTENANCE_GUIDE.md # 维护指南（本文档）
└── HELP_CENTER_CHANGELOG.md      # 更新日志
```

---

### 内容管理流程

#### 1. 内容创建

```
1. 在 help-articles.ts 中添加文章数据
2. 设置文章状态为 'draft'
3. 编写文章内容
4. 自我审查
5. 提交审核
6. 状态改为 'published'
```

#### 2. 内容更新

```
1. 定位要更新的文章
2. 修改内容
3. 更新 updatedAt 字段
4. 更新 version 字段
5. 记录变更日志
```

#### 3. 内容归档

```
1. 确认文章已过时
2. 状态改为 'archived'
3. 保留历史记录
4. 创建新的替代文章
```

---

### 版本控制

#### 版本号规则

格式：`v主版本。次版本.修订版`

- **主版本**：重大更新（新功能、架构变化）
- **次版本**：功能更新（新增文章、功能增强）
- **修订版**：小幅更新（内容修正、Bug 修复）

**示例**：
- `v3.2.3` → 当前版本
- `v3.2.4` → 新增文章
- `v3.3.0` → 新增功能
- `v4.0.0` → 重大更新

---

#### 变更日志

每次更新都应记录变更日志：

## 📊 数据分析

### 收集指标

#### 使用数据

- **文章访问量**：每篇文章的访问次数
- **搜索关键词**：用户搜索的热门关键词
- **停留时间**：用户在文章页面的停留时间
- **收藏次数**：文章被收藏的次数
- **反馈评分**：用户对文章的评分

---

#### 用户行为

- **访问路径**：用户的浏览路径
- **跳出率**：只访问一篇文章就离开的比例
- **转化率**：阅读后问题得到解决的比例
- **回访率**：用户再次访问的比例

---

### 分析方法

#### 周报

每周一份数据报告：
- 热门文章 TOP 10
- 搜索关键词 TOP 10
- 用户反馈汇总
- 改进建议

---

#### 月报

每月一份深度分析：
- 内容质量趋势
- 用户满意度变化
- 功能使用情况
- 优化建议
